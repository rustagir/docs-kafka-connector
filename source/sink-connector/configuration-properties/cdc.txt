Overview
--------

.. _sink-configuration-change-data-capture-description-start:

Use the following configuration settings to specify a class the sink
connector uses to process change data capture (CDC) events.

.. _sink-configuration-change-data-capture-description-end:

.. include:: /includes/sink-config-link.rst

Settings
--------

.. _sink-configuration-change-data-capture-table-start:

.. list-table::
   :header-rows: 1
   :widths: 25 75

   * - Name
     - Description

   * - | **change.data.capture.handler**
     - | **Type:** string
       |
       | **Description:**
       | The class name of the CDC handler to use for converting changes
         into event streams.
       |
       | **Default**: ``""``
       | **Accepted Values**:
       | The following handlers are built into the
         sink connector:
       | - ``"com.mongodb.kafka.connect.sink.cdc.debezium.mongodb.MongoDbHandler"``
       | - ``"com.mongodb.kafka.connect.sink.cdc.mongodb.ChangeStreamHandler"``
       | - ``"com.mongodb.kafka.connect.sink.cdc.qlik.rdbms.RdbmsHandler"``

       .. tip::

          To learn more about the built-in handlers, see the guide on
          :ref:`Sink Connector Change Data Capture Handlers
          <sink-fundamentals-cdc-handler>`.
          
       | You can also use an empty string or a fully qualified Java
         class name.
